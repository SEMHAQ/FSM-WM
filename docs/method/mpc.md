# MPC 集成

## 什么是MPC（模型预测控制）？

想象你在开车。你不会只看眼前——你会往前看一段路，预判"如果我这样打方向盘，车会怎么走"，然后选最优的动作。

**MPC就是这个"往前看"的过程**：

```
当前状态
    ↓
世界模型预测：如果我施加动作序列 [a1, a2, ..., a10]，未来10步会怎样？
    ↓
优化器：反复调整动作序列，让预测的轨迹尽量接近目标
    ↓
执行：只执行第一步动作，然后重新规划
    ↓
循环
```

## 世界模型在MPC中的角色

MPC需要**大量**前向预测——每次规划都要反复问世界模型："如果这样做会怎样？"

- LSTM做世界模型 → 每次预测27.8ms → MPC太慢
- SSM-WM做世界模型 → 每次预测3.8ms → MPC实时可行

## 为什么用FFT卷积模式？

MPC的内层优化需要做50次Adam迭代，每次都涉及对同一时刻的动作序列进行前向传播。

- 递推模式：一步一步算，50次就是50次串行计算
- 卷积模式：一次性算完所有时间步，50次并行处理

**卷积模式在MPC场景下效率高得多**，这就是为什么推理时间实验都在卷积模式下测量。

## 控制频率

| 数据集 | SSM-WM-MPC频率 | LSTM-MPC频率 | 加速比 |
|--------|---------------|-------------|--------|
| 合成数据集 | 5.1Hz | 0.7Hz | **7倍** |
| MuJoCo Humanoid | 2.1Hz | 0.35Hz | **6倍** |

5.1Hz意味着每秒可以做5次完整的MPC规划，满足大多数运动规划需求。
