# 背景概念

## 什么是世界模型？

想象你在学骑自行车。你不需要每次都真的骑上去摔一跤才知道"往左打太多会倒"——你的大脑里有一个**心理模型**，能模拟"如果我这样动，会发生什么"。

**世界模型**就是给机器人建的这个"心理模型"。

```
机器人看到当前状态（关节角度、角速度...）
    ↓
世界模型预测：如果我施加这个力矩，下一刻身体会怎样？
    ↓
MPC控制器：用这个预测能力，搜索最优的动作序列
```

没有世界模型 → 机器人只能"试错"
有世界模型   → 机器人可以"想象"，提前规划

## 为什么需要"序列模型"？

机器人的状态是随时间变化的序列数据。要预测"下一步"，就需要理解"前面每一步"的模式。

**序列模型的任务**：输入一段历史序列，输出对未来的预测。

## 四种序列模型对比

### LSTM — 经典选手

- 原理：用"门"控制信息的保留和遗忘，一步步往前推
- 优点：好理解，非线性能力强
- 缺点：**慢**，必须一步一步算，无法并行。T=64时推理要27.8ms

### Transformer — 注意力之王

- 原理：让序列中每个位置都能"看到"其他所有位置，计算注意力权重
- 优点：建模能力最强
- 缺点：**更慢**，计算量是序列长度的平方 O(T²)。T=64时推理超过100ms

### SSM（状态空间模型）— 我们用的

- 原理：用线性微分方程描述状态变化，数学上和控制理论里的"线性系统"是同一个东西
- 优点：可以并行计算（FFT卷积），复杂度 O(T log T)。推理只要3.8ms
- 缺点：线性假设在某些复杂场景下有局限

### Mamba — SSM的热门变体

- 原理：在SSM基础上加了"选择性扫描"，让模型能根据输入动态调整状态更新
- 优点：精度好
- 缺点：需要自定义CUDA算子，**部署门槛高**

## SSM vs Mamba 的关键区别

| | SSM（我们用的） | Mamba |
|---|---|---|
| 数学基础 | 相同 | 相同 |
| 实现方式 | 标准PyTorch（FFT） | 自定义CUDA算子 |
| 部署难度 | **低** | 高（需要编译mamba-ssm库） |
| 精度 | 略低（MuJoCo差1.5%） | 略高 |
| 推理速度 | **更快**（3.8ms vs 4.5ms） | 稍慢 |

**我们的选择**：用更简单的SSM实现，牺牲1.5%的精度，换来更好的部署友好性。
