# 实验设置

## 两个数据集，两个目的

| | 合成数据集 | MuJoCo Humanoid |
|---|---|---|
| **目的** | 验证计算效率和MPC | 验证预测精度 |
| **动力学** | 近线性（简单） | 真实接触+摩擦+碰撞（复杂） |
| **状态维度** | 376 | 376 |
| **动作维度** | 17 | 17 |
| **数据量** | 500条episode | 200条episode |

**为什么用两个？** 因为速度优势和精度优势可能来自不同场景。用两个数据集分别验证，比一个数据集同时验证更严谨。

## 对比方法

- **LSTM-WM**：经典基线，2层和4层版本
- **Transformer-WM**：注意力机制，精度好但慢
- **Mamba-WM**：SSM的热门变体，需要CUDA算子
- **线性回归**：最简单的baseline，验证非线性建模的必要性

## 评价指标

- **MSE**：预测误差，越小越好（单位 ×10⁻³）
- **R²**：模型解释了多少方差，越接近1越好
- **推理时间**：越快越好（ms）
- **参数量**：越少越好（M = 百万）
