# 消融实验

## 核心问题：每个组件到底贡献了多少？

消融实验就是"拆零件"——把模型的一个组件去掉，看精度掉多少。

## 门控机制：贡献最大

| 去掉什么 | 合成数据集MSE变化 | MuJoCo MSE变化 |
|---------|----------------|---------------|
| 去掉门控 | +2.2% | +4.6% |
| 去掉残差连接 | +1.5% | +2.6% |

门控在MuJoCo上贡献更大——因为接触动力学有更多"不连续"，需要门控来选择性过滤信息。

## 网络深度和宽度

| 配置 | MSE | 说明 |
|------|-----|------|
| L=2层 | 2.99 | 太浅，建模能力不足 |
| **L=4层（默认）** | **2.72** | 平衡点 |
| L=6层 | 2.64 | 再加层收益很小 |
| D=64 | 2.93 | 太窄 |
| **D=128（默认）** | **2.72** | 平衡点 |
| D=256 | 2.58 | 参数量增3.5倍，精度只改善5% |

**结论：L=4, N=16, D=128是最佳平衡点。**

## 门控阈值：软阈值最优

| 阈值类型 | MSE | 说明 |
|---------|-----|------|
| **软阈值（sigmoid）** | **0.834** | 连续输出，适应渐变 |
| Garrote | 0.852 | 稀疏但连续 |
| 硬阈值（0/1） | 0.891 | 二值信号，无法表达连续变化 |

硬阈值差6.4%——因为接触力是渐变的，不是"有或没有"。

## 训练损失：多步损失很关键

| λ | 单步MSE | 多步MSE (H=8) |
|---|---------|--------------|
| 0（仅单步） | 2.64 | 9.99 |
| **0.5（默认）** | **2.72** | **7.17** |
| 1.0 | 2.84 | 7.05 |

λ=0.5在单步精度和多步稳定性之间取得最佳平衡。多步损失让模型学会"自己的误差会怎么累积"。
