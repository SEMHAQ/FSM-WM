# 讨论

## 核心论点：精度不是唯一标准

SSM-WM的价值不在于"比所有方法都准"，而在于**精度-速度-参数量的综合权衡**。

```
精度排名：Mamba ≈ SSM-WM > LSTM > Transformer
速度排名：SSM-WM > Mamba > LSTM >> Transformer
参数排名：SSM-WM < Mamba < LSTM < Transformer
```

综合来看，SSM-WM的"性价比"最高。

## 合成数据集 vs MuJoCo：为什么表现不同？

| | 合成数据集 | MuJoCo |
|---|---|---|
| 动力学特点 | 近线性（系数0.1） | 真实接触+碰撞 |
| SSM-WM排名 | 最慢（但最快） | **优于LSTM和Transformer** |
| 原因 | 线性假设在简单场景反而不利 | 结构化参数化在复杂场景更稳定 |

**关键洞察**：不能只看一个数据集的结果。两个数据集的互补设计，让我们对SSM-WM的能力有了更全面的认识。

## 和Mamba的对比：1.5%的精度差距值不值？

| 维度 | SSM-WM | Mamba-WM | 差距 |
|------|--------|----------|------|
| MuJoCo MSE | 0.834 | 0.821 | 1.5%（不显著） |
| 推理速度 | 3.8ms | 4.5ms | SSM快16% |
| 参数量 | 0.24M | 0.28M | SSM少14% |
| 实现 | 标准PyTorch | 自定义CUDA | SSM更简单 |

**结论**：为了1.5%的精度（统计不显著），值得用更复杂的实现吗？在实际部署场景下，答案通常是"不值得"。
